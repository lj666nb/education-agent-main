# A3-基于大模型的个性化资源生成与学习多智能体系统开发

## 核心创新点：动态学习画像维护

### 画像维度设计（超越赛题要求）

除了赛题要求的6个维度（知识基础、认知风格、易错点偏好等），建议增加：

- **学习活跃时段**：捕捉学生最高效的学习时间窗口
- **多模态偏好**：向量化表征，如视频偏好 vs 文档偏好
- **遗忘曲线参数**：针对不同知识点的个性化记忆留存率
- **元认知校准度**：对比自评信心与实际表现，区分“真掌握”与“假掌握”

### 双层存储架构

**长效记忆（Long-term）**：向量数据库（Milvus/pgvector）+ 图数据库（Neo4j）
- 存储稳定特征：多模态偏好向量、知识点掌握度分数与置信度
- 图结构：学生节点、知识点节点、掌握关系边（含分数与置信度）

**短时缓存（Short-term）**：Redis Hash
- 存储当前会话状态：正在学习的模块、最近3次答题正误率、认知负荷状态
- 自动TTL过期，防止数据残留

### 秒级动态微调机制

**核心架构**：事件总线 + 画像更新智能体（Profile Agent）

| 组件 | 职责 |
|------|------|
| 前端埋点SDK | 无痕采集视频（播放/暂停/退出）、答题（选择/修改/提交）、页面（失焦/空闲）等事件 |
| 消息队列（Kafka） | 异步传输事件，解耦采集与处理 |
| Profile Agent | 消费事件，上下文增强，信号计算与累积，阈值触发画像更新及资源预生成任务 |
| 信号累积（Redis Sorted Set） | 按信号类型与目标知识点累积权重，24小时TTL防止陈旧信号滞留 |

**关键规则**：
- 不直接覆盖画像，采用信号累积机制：如“视频中途退出数学内容”累积薄弱点信号（权重+5），超过阈值15才正式更新Neo4j中的掌握度分数
- 触发后自动向Task Queue发布资源生成任务（如：为该生预生成一份“梯度消失的图解版文档”）
- 维护完整的可解释性日志时间线，供前端可视化展示
