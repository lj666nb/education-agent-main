## 1. 概述

### 1.1 功能目标

设计一套完整的用户注册与登录系统，充分考虑用户各种非法操作场景，以清晰的中文提示让用户立即知道如何操作。

### 1.2 用户角色

| 角色 | 描述 |
|------|------|
| 访客 | 未注册用户，可注册和登录 |
| 学生 | 已注册并登录的普通用户 |
| 管理员 | 系统管理员，可管理用户 |

---

## 2. 注册功能

### 2.1 字段规范

| 字段 | 类型 | 必填 | 约束 | 错误提示 |
|------|------|------|------|----------|
| 用户名 | 字符串 | 是 | 3-50字符，仅字母/数字/下划线 | 「用户名长度3-50位，仅支持字母、数字和下划线」 |
| 密码 | 字符串 | 是 | 最少6位，最多100位 | 「密码长度至少6位」 |
| 确认密码 | 字符串 | 是 | 与密码一致 | 「两次输入的密码不一致」 |
| 邮箱 | 字符串 | 否 | 有效的邮箱格式 | 「请输入有效的邮箱地址」 |
| 专业 | 字符串 | 是 | 1-200字符 | 「请输入您的专业」 |

### 2.2 注册流程

```
访客 → 填写注册表单 → 前端预校验 → 提交注册
                                        ↓
                              ┌─────────┴─────────┐
                              ↓                   ↓
                          注册成功            注册失败
                              ↓                   ↓
                          跳转登录页        显示错误提示
```

### 2.3 边界情况处理

#### 3.1 用户名校验

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 用户名为空 | 400 | 「请输入用户名」 |
| 用户名长度<3 | 400 | 「用户名长度3-50位」 |
| 用户名长度>50 | 400 | 「用户名长度3-50位」 |
| 包含非法字符 | 400 | 「用户名仅支持字母、数字和下划线」 |
| 用户名已存在（正常账号） | 400 | 「该用户名已被注册，请使用其他用户名」 |
| 用户名已存在（已注销账号） | 201 | 「账号已重新激活」 |

#### 3.2 密码校验

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 密码为空 | 400 | 「请输入密码」 |
| 密码长度<6 | 400 | 「密码长度至少6位」 |
| 密码长度>100 | 400 | 「密码长度最多100位」 |
| 确认密码为空 | 400 | 「请确认密码」 |
| 确认密码与密码不一致 | 400 | 「两次输入的密码不一致」 |

#### 3.3 邮箱校验

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 邮箱格式无效 | 400 | 「请输入有效的邮箱地址」 |
| 邮箱已注册（其他账号） | 400 | 「该邮箱已被注册，可使用邮箱登录」 |

#### 3.4 专业校验

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 专业为空 | 400 | 「请输入您的专业」 |
| 专业长度>200 | 400 | 「专业名称过长」 |

#### 3.5 其他异常

| 场景 | 响应码 | 中文提示 |

| 服务器内部错误 | 500 | 「注册失败，请联系技术支持」 |

## 3. 登录功能

### 3.1 字段规范

| 字段 | 类型 | 必填 | 约束 | 错误提示 |
|------|------|------|------|----------|
| 用户名 | 字符串 | 是 | 已注册的用户名 | 「请输入用户名」 |
| 密码 | 字符串 | 是 | 正确的密码 | 「请输入密码」 |

### 3.2 登录流程

```
访客 → 填写登录表单 → 前端预校验 → 提交登录
                                        ↓
                              ┌─────────┴─────────┐
                              ↓                   ↓
                          登录成功            登录失败
                              ↓                   ↓
                      写入Token，跳转         显示错误提示
                      个人中心/首页
```

### 3.3 边界情况处理

#### 3.3.1 账号锁定（暴力破解防护）

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 连续5次密码错误 | 429 | 「登录失败次数过多，账号已锁定。请15分钟后重试」 |
| 锁定期间尝试登录 | 429 | 「账号已锁定，请{剩余时间}分钟后重试」 |

#### 3.3.2 用户名校验

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 用户名为空 | 400 | 「请输入用户名」 |
| 用户名不存在 | 401 | 「该账号未注册，请先注册」 |
| 用户名已注销 | 401 | 「该账号未注册，请先注册」 |
| 用户名已停用 | 403 | 「账号已停用，请联系管理员」 |

#### 3.3.3 密码校验

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| 密码为空 | 400 | 「请输入密码」 |
| 密码错误（剩余N次） | 401 | 「密码错误，还可以尝试{5-N}次」 |
| 密码错误（最后1次） | 401 | 「密码错误，这是最后一次尝试机会」 |

#### 3.3.4 Token处理

| 场景 | 响应码 | 中文提示 |
|------|--------|----------|
| Token过期 | 401 | 「登录已过期，请重新登录」 |
| RefreshToken无效 | 401 | 「登录已过期，请重新登录」 |
| RefreshToken已使用 | 401 | 「登录异常，请重新登录」 |

## 5. 用户反馈规范

> **原则**：任何操作（成功/失败）都必须即时反馈给用户，让用户清楚知道发生了什么以及接下来该怎么做。

### 5.1 反馈类型

| 类型 | 样式 | 持续时间 | 适用场景 |
|------|------|----------|----------|
| 成功提示 | 绿色背景 + ✓ 图标 | 3秒后自动消失 | 操作成功 |
| 错误提示 | 红色背景 + ✕ 图标 | 手动关闭或5秒后消失 | 操作失败 |
| 警告提示 | 黄色背景 + ⚠ 图标 | 手动关闭 | 需要用户注意 |
| 加载中 | 旋转图标 + 文字 | 直到完成 | 异步操作中 |
| Toast 提示 | 顶部居中弹出 | 3秒后自动消失 | 轻量级反馈 |

### 5.2 反馈消息规范

#### 5.2.1 操作结果反馈（前端表单操作）

| 操作 | 成功反馈 | 失败反馈 |
|------|----------|----------|
| 注册 | 「注册成功！正在跳转登录页...」 | 显示后端返回的 `detail` 错误信息 |
| 登录 | 「登录成功！正在跳转...」 | 显示后端返回的 `detail` 错误信息 |
| 登出 | 「已安全退出」 | 「退出失败，请重试」 |
| 修改密码 | 「密码修改成功」 | 显示具体错误原因 |
| 注销账号 | 「账号已注销，数据将保留30天」 | 「注销失败，请联系管理员」 |
| 更新资料 | 「资料更新成功」 | 显示具体错误原因 |

#### 5.2.2 API 错误反馈（HTTP 状态码）

| 状态码 | 场景 | 用户提示 |
|--------|------|----------|
| 400 | 请求参数错误 | 后端返回的 `detail` 信息 |
| 401 | 未认证/Token无效 | 「登录已过期，请重新登录」 |
| 403 | 无权限访问 | 「您没有权限执行此操作」 |
| 404 | 资源不存在 | 「请求的内容不存在」 |
| 422 | 数据验证失败 | 后端返回的验证错误信息（中文） |
| 429 | 请求过于频繁 | 「操作过于频繁，请稍后再试」 |
| 500 | 服务器内部错误 | 「服务暂时不可用，请稍后重试」 |
| 503 | 服务不可用 | 「服务维护中，请稍后访问」 |

### 5.3 前端错误处理代码示例

```typescript
// 标准 API 调用错误处理
try {
  await authApi.register({ username, password, major })
  showToast('注册成功！正在跳转登录页...', 'success')
  navigate('/login')
} catch (err: any) {
  const status = err.response?.status
  const detail = err.response?.data?.detail

  // 记录错误日志（生产环境）
  console.error('注册失败:', { status, detail, err })

  if (status === 400) {
    showToast(detail || '注册信息有误', 'error')
  } else if (status === 422) {
    showToast(detail || '表单填写有误', 'error')
  } else if (status === 500) {
    showToast('服务器错误，请稍后重试', 'error')
  } else {
    showToast(detail || '注册失败，请检查网络', 'error')
  }
}
```

### 5.4 Toast 组件规范

## 6. 安全设计

### 6.1 暴力破解防护

- 同一用户名连续5次登录失败，锁定15分钟
- 前端登录按钮点击后添加loading状态，防止重复提交
- 前端添加图形验证码（未来扩展）

### 6.2 密码安全

- 密码前端不存储，仅传输
- 密码使用bcrypt加密存储
- 不返回密码字段给前端

### 5.3 Session管理

| 策略 | 说明 |
|------|------|
| AccessToken | 有效期24小时 |
| RefreshToken | 有效期7天 |
| 单设备登录 | 支持（可扩展） |
| 登出清理 | 删除服务端session缓存 |

## 7. 状态码规范

### 7.1 HTTP状态码

| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 登录成功、获取信息成功 |
| 201 | 创建成功 | 注册成功 |
| 400 | 请求错误 | 参数校验失败、字段格式错误 |
| 401 | 未授权 | 密码错误、未登录、Token无效 |
| 403 | 禁止访问 | 账号已停用 |
| 429 | 请求过多 | 登录失败次数过多被锁定 |
| 500 | 服务器错误 | 数据库错误、系统异常 |

### 7.2 错误响应格式

```json
{
  "detail": "错误描述（中文）",
  "code": "ERROR_CODE",
  "extra": {}
}
```
