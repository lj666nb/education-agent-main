# 如何做好前端设计：从 Claude Design 挖掘出的高品质实践

> 本指南源于 Anthropic 的 Claude Design 及其对应 Skill 的核心思想，将 AI 辅助设计流程提炼为可供开发者与设计师直接应用的前端实践原则。无论你是否使用 AI 工具，这些方法都能有效提升网页设计的专业质感。

---

## 1. 角色定位：把自己当产品经理，把 AI 当专家设计师

传统开发中，开发者常同时扮演设计与执行的角色，容易陷入“能用就行”的妥协。高质量前端设计的第一步，是明确分工：

- **你是产品经理**：负责定义目标、用户需求、内容策略和验收标准。
- **AI（或你的设计系统）是专家设计师**：它应按任务动态切换身份——动效设计师、UX 架构师、视觉排版专家等。

这样角色分离后，设计决策会更果断，不再因开发者个人审美局限而产出同质化页面。

**实践建议**：
- 在团队中，让设计评审成为独立环节，即便是个人项目，也用“设计稿 → 代码”的流程约束自己。
- 若使用 AI 辅助，先清晰描述产品目标与用户场景，再让 AI 输出方案，避免直接让它“写个页面”。

---

## 2. 工作流定义：六步闭环，避免冗长复述

Claude Design 的成功很大程度取决于其严格的工作流，我们可以抽象为前端开发的标准流程：

1. **需求理解**：AI 或设计师按需提问，明确核心目标，而非立即动手。
2. **设计策略**：输出整体风格、布局方向与组件选型，不涉及细节。
3. **关键注意事项**：仅列出会影响实现的关键约束（如无障碍、性能风险、品牌规范）。
4. **下一步动作**：明确告知“现在该做什么”，比如“先完成导航栏的响应式骨架”。
5. **执行与编码**：遵循既定策略输出可运行代码，保持最小可用版本思维。
6. **递进优化**：基于整体评审，逐轮增强，而非一次性堆砌细节。

**核心原则**：每一步只输出该步所需的最小信息，杜绝复述需求、解释常识或提前展示结果。这能保持上下文干净，让开发者与设计师都聚焦于当前决策。

---

## 3. 去除“AI 味”：告别廉价设计的视觉雷区

很多快速生成的前端页面一眼就能被认出，因为它们共享一套烂大街的默认美学。你需要主动禁止以下雷区：

- ❌ 俗套渐变（如蓝紫、彩虹渐变）
- ❌ Emoji 作为图标
- ❌ 过度使用的字体（Arial、Comic Sans 等）
- ❌ 无意义的大数字、假数据仪表盘
- ❌ 冗余的阴影与边框装饰

**替代方案**：
- 优先使用纯色、微妙的噪点纹理或网格背景。
- 图标采用专业图标库（Lucide、Phosphor Icons），并统一风格与描边宽度。
- 字体推荐高品质现代选择，如 **Inter、Manrope、Satoshi、DM Sans、Instrument Sans**，并根据氛围搭配衬线字体（如 **Playfair Display、Lora**）。

记住：一个页面看起来“专业”，往往是因为它克制地使用了视觉元素。

## 4. 色彩系统：基于 OKLCH 的和谐派生

无规律的配色是页面混乱的主要来源。做好前端设计，必须建立科学的色彩派生逻辑：

1. **确立品牌主色**：从品牌 Logo 或产品气质出发，定义 1-2 个核心颜色。
2. **使用 OKLCH 色彩空间派生**：  
   用 `oklch()` 调整亮度（Lightness）、色度（Chroma）与色相（Hue），能与人眼感知更匹配，避免传统 HSL/RGB 带来的亮度不均匀问题。
   - **浅色模式**：提高亮度，略微降低色度，形成背景、卡片等层次。
   - **深色模式**：大幅降低亮度，微调色度以保持色彩辨识。
   - **功能色**：保持色相稳定，仅调整亮度/色度生成错误、成功、警告状态色。
3. **避免凭空配色**：所有辅助色均应从主色派生，或选用主色的邻近色、互补色，确保全局和谐。

**CSS 实践**：在 `:root` 中定义基于 OKLCH 的设计令牌（Design Tokens），组件只引用语义化变量，不直接使用裸色值。

## 5. 内容原则：极简主义，每个元素都有存在的理由

高品质前端设计一定不是内容的堆砌，而是空间的经营。

- **极简思维**：每个元素、每行文字都必须回答“它为什么在这里”。如果去掉某元素页面理解不受影响，就该果断删除。
- **用留白解决空旷**：页面看起来空，不是内容少，而是疏密关系没处理好。增加段落间距、内边距、模块间距，让留白成为组织信息的主动手段，而不是被动缺内容的空白。
- **排版优先**：通过字号对比、字重变化、行高节奏建立清晰的信息层级。一级标题与正文的字号比可达到 2.5~3 倍，而非传统的 1.5 倍，这能立即提升设计感。

实现时，先只写语义正确的 HTML，完全不用 CSS，检查信息结构是否清晰。然后逐步添加最少的样式，始终保持内容的主导权。

## 6. 验证机制：用独立审查打破“自我感觉良好”

开发者对自己刚做完的页面容易产生盲区——觉得对齐完美、配色舒适，但实际上可能存在明显缺陷。

- **引入独立子 Agent 或同行评审**：完成初版后，使用一个独立的、专门检查设计质量的 Agent（或邀请另一位设计师）进行审查。审查目标应包含：无障碍合规、色彩对比度、间距一致性、移动端适配、字体回退策略等。
- **建立检查清单**：
  - 所有可交互元素是否有明确的 `:focus-visible` 样式？
  - 文本对比度是否满足 WCAG AA（4.5:1 以上）？
  - 200% 缩放时布局是否不丢失信息？
  - 图片是否有合理的 `alt` 文本？
  - 无 JavaScript 时关键内容是否仍可访问？

这个独立验证步骤能极大减少因主观偏好造成的设计事故。

---

## 7. 从工具到 Skill：将规范可执行化

上述原则在手工项目中容易松懈，更好的方式是将它们封装为可复用的开发规范或 Skill。

- **设计系统先行**：在写任何组件代码前，先定义色彩令牌、字体比例、间距刻度、圆角阶梯和阴影层级。可用一个单独的 `design-tokens.css` 或 `theme.config.ts` 集中管理。
- **尽早产出最小可用版本**：先做一个仅有核心功能与布局的灰阶原型（灰度模式），验证交互流程与信息架构，之后再应用具体视觉风格。
- **Skill 化（AI 场景）**：若使用 Cursor、Claude Code 等开发工具，可将上述要点制作成 `.cursorrules` 或项目 Skill 文件，让 AI 自动在每次生成代码时遵循色彩、排版、交互与审查规范。

## 8. 效果对比：专业质感从何而来

同样的产品需求，缺乏以上原则产出的页面通常表现为：
- 配色艳俗或杂乱
- 字体默认为系统无衬线，缺乏个性
- 布局居中盒子加阴影，像模板套用
- 信息平铺，没有节奏

而应用了色彩派生、审慎选字、留白经营与独立审查的页面，会呈现：
- 色彩和谐，主次分明，深色/浅色模式切换自然
- 字体组合独特，正文可读性极佳，标题具有品牌识别度
- 布局富有创意，不对称分割、重叠效果、微动效都服务于内容
- 整体质感逼近专业设计师手工打磨的作品

这种差距无关代码能力，只关乎 **是否遵守了经过验证的设计约束**。

**做好前端设计，不是会写更多 CSS 技巧，而是建立一套可执行的、克制且科学的设计纪律。**  
将这些原则融入你的开发习惯或工具链，每一个页面都能成为你能力的代表作。
